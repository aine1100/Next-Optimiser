# CI/CD Integration

## Basic CI Gate

```bash
npx next-optimize ci --threshold 80
```

## Regression Detection

```bash
# Establish baseline (once)
npx next-optimize baseline save

# CI — fail only on regression
npx next-optimize ci --threshold 70 --baseline --save-baseline
```

## GitHub Actions (Full Example)

```yaml
name: Performance Audit
on: [pull_request]

jobs:
  audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm run build

      - name: Restore baseline
        uses: actions/cache@v4
        with:
          path: .next-optimize-baseline.json
          key: perf-baseline-${{ github.base_ref }}

      - name: Performance Audit
        run: npx next-optimize ci --threshold 80 --baseline --save-baseline

      - name: Upload SARIF
        if: always()
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: next-optimize-results.sarif

      - name: Upload HTML Report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: performance-report
          path: next-optimize-report.html
```

## Threshold Strategies

| Strategy | Threshold | Use Case |
|----------|-----------|----------|
| Incremental | 50 → 90 over time | Legacy apps |
| Strict | 90+ | New projects |
| Regression-only | `--baseline` | Mature teams |
