# Contributing to Next Optimize Platform

Thank you for your interest in contributing to the Next Optimize Platform! We welcome all contributions, including bug reports, feature requests, documentation improvements, and pull requests.

## How to Contribute

### 1. Reporting Bugs
- Use the GitHub Issues tracker to report bugs.
- Include a clear description of the problem, steps to reproduce, and your environment (OS, Node version, project framework).

### 2. Feature Requests
- Open an issue with the "feature-request" label.
- Describe the use case and the proposed solution.

### 3. Pull Requests
- Fork the repository and create a new branch from `main`.
- Ensure your code follows the existing style (TypeScript strict mode, JSDoc comments).
- Run `npm run build` to ensure no compilation errors.
- Include a descriptive PR message and link to any relevant issues.

## Development Setup

1. Fork and clone the repository.
2. Install dependencies:
   ```bash
   npm install
   ```
3. Run in development mode:
   ```bash
   npm run dev analyze
   ```

## Documentation Guidelines
- All public APIs (Classes, Methods, Interfaces) must have JSDoc comments.
- Keep specialized guides in the `docs/` folder.
- Ensure the `README.md` is updated for any new user-facing features.

## Community & Code of Conduct
Please be respectful and professional in all interactions. We follow the standard Contributor Covenant Code of Conduct.
